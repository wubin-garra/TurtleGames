# Prediction Globe - 3D 预测市场地球可视化

## 概述

一个类似 Google Earth 的 3D 交互式地球页面，在地球表面展示 Polymarket 预测市场的热点事件。每个事件根据地理相关性定位在地球上，通过发光标记点、光柱等视觉效果呈现，用户可以悬停预览和点击查看详情。

**定位**: 个人技术展示项目，作为作品集的一部分。

## 技术栈

- 纯 HTML + JS 单页面（`prediction-globe.html`）
- Three.js（CDN 引入）用于 3D 地球渲染
- Polymarket Gamma API（公开，无需认证）获取数据
- 无构建工具、无框架依赖

## 数据源

### Polymarket Gamma API

- **Base URL**: `https://gamma-api.polymarket.com`
- **端点**: `/events`（按 volume 排序获取热门事件）
- **认证**: 无需认证，完全公开
- **请求量**: 每次取 30-50 条事件，每 5 分钟刷新

### 数据字段（预期使用）

- 事件标题、描述
- 当前概率（Yes/No）
- 交易量（volume）
- 流动性
- 分类标签
- 事件 URL（用于外链跳转）

## 3D 地球渲染

### 球体

- Three.js SphereGeometry + NASA 夜间灯光纹理
- 大气层光晕效果（自定义 shader，蓝色边缘发光）
- 自动缓慢旋转
- OrbitControls 支持拖拽旋转、滚轮缩放

### 背景

- 深色太空背景
- 星空粒子系统（随机分布小点）

## 标记点设计

### 视觉映射

| 属性 | 映射规则 |
|------|---------|
| 大小 | 按交易量（volume）映射，热门事件更大 |
| 颜色 | 按概率映射：绿色=大概率 Yes，红色=大概率 No，黄色=50/50 |
| 光柱高度 | 按交易量映射 |
| 动画 | 持续脉冲涟漪效果 |

### 渲染方式

- 使用 InstancedMesh 批量渲染，上限 50 个标记点
- 每个标记点 = 发光圆点 + 向上延伸的细光柱

## 地理定位

### 关键词匹配

- 内置 ~50 个国家/地区的经纬度映射表
- 从事件标题/描述提取关键词匹配地理位置
- 示例：
  - `"Trump"` / `"US"` / `"America"` → `[38.9, -77.0]`（华盛顿 DC）
  - `"Ukraine"` / `"Zelensky"` → `[50.4, 30.5]`（基辅）
  - `"Bitcoin"` / `"Ethereum"` → 全球类，均匀分散

### 防重叠

- 同一地区多个事件时，在该位置周围小范围随机偏移
- 无法匹配地理位置的事件归为"全球"类，均匀分散在地球上

## 交互设计

### 悬停卡片（Tooltip）

- 鼠标悬停标记点 → 旁边浮出半透明卡片
- 内容：标题、当前概率（大字）、24h 变化、交易量
- 毛玻璃效果（`backdrop-filter: blur`）+ 发光边框

### 侧边栏详情面板

- 点击标记点 → 地球自动旋转居中到该位置 → 右侧滑出详情面板
- 面板内容：
  - 事件标题 + 描述
  - 概率柱状图（Yes/No）
  - 交易量、流动性
  - "在 Polymarket 查看" 外链按钮
- 点击空白处或 X 按钮关闭面板

### 地球交互

- 拖拽旋转
- 滚轮缩放
- 点击标记时自动旋转到对应位置

## UI 布局

### 顶部控制栏

- 左上角：标题 "Prediction Globe"
- 右上角：分类筛选按钮（政治/加密/体育/科技）+ 搜索框
- 筛选时不匹配的标记点淡出，匹配的保持高亮

### 底部信息栏

- 当前展示事件数量
- 数据更新时间
- "Powered by Polymarket" 标注

### 响应式布局

- **桌面端**: 地球占 70% 宽度，侧边栏 30%
- **移动端**: 地球全屏，卡片和详情面板从底部滑上来

## 数据更新策略

- 首次加载时获取数据
- 每 5 分钟自动刷新概率和交易量
- 刷新时标记点平滑过渡（大小/颜色渐变），不闪烁
- API 失败时显示 localStorage 缓存数据 + 提示"数据可能不是最新"

## 性能优化

- 标记点上限 50 个
- InstancedMesh 批量渲染
- 纹理懒加载：先显示简单线框地球，纹理加载完后淡入
- Raycaster 节流（悬停检测限制频率）
