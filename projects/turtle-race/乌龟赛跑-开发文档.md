## 1. 项目概览

一款面向儿童的赛跑博弈游戏。4只乌龟同场竞技，玩家选择一只下注并观看比赛。核心玩法循环：

```
主页 → 选龟 → 选赛道 → 下注 → 比赛(3圈) → 结算 → 循环
            ↘ 段位赛(无下注，按段位收费/奖励)
```

### 核心系统
- **货币**：虚拟比特币(BTC)，初始 9999
- **乌龟收集**：9只乌龟，4只初始解锁，5只商店购买
- **装备**：4个部位(头饰/面饰/背饰/坐骑)，60+件装备
- **套装**：5套传说/神话级套装，4件套触发主动技能
- **鞋子**：6种地形专属鞋
- **赛道**：7条赛道(含铁人三项和彩虹)
- **段位**：7级段位(青铜→王者)，Bo5晋级/降级
- **道具**：6种比赛内消耗品

## 2. 技术架构

### 渲染模式
- **静态 UI**：`render()` 函数更新 DOM，由事件触发调用
- **比赛动画**：`requestAnimationFrame` 驱动的 `tick()` 循环，每帧调用 `render()` 更新位置
- **无框架**：原生 JS + DOM 操作，无 React/Vue

### 场景切换
```javascript
function showScene(id) {
  document.querySelectorAll('.scene').forEach(s => s.classList.remove('active'));
  document.getElementById(id).classList.add('active');
}
```

### 数据流
```
localStorage ← saveGame() ← state → render() → DOM
                                   → tick() (比赛中)
```

## 5. HTML 场景结构

6个场景 `<div class="scene">`，通过 `.active` 类切换显示：

### scene-start（主页）
- Logo + 标题 + 动画背景粒子
- 按钮：开始比赛、段位赛、商店、排行榜、规则、拜财神
- 显示余额和最高分

### scene-race（比赛场景）
- **顶栏**：返回、Logo、圈数、规则、静音
- **段位横幅** `#rankedRaceBanner`：段位赛时显示段位信息
- **赛道区域**：
  - 赛道标题 `#raceTitle`
  - 赛道容器：起跑线、4车道、终点线
  - 赔率显示 `#raceOdds`、连胜 `#streakDisplay`
- **底部栏**：
  - 余额 `#coins`
  - 下注选择 `#betBtns`（排位赛时隐藏）
  - 道具按钮 `#itemBtns`（含套装技能按钮）
  - 选手按钮 `#pickBtns`
  - 开始按钮 `#startBtn`

### scene-shop（商店）
- 分类标签：全部/乌龟/头饰/面饰/背饰/坐骑/鞋子/套装
- 动态渲染商品卡片

### scene-leaderboard（排行榜）
- 动态渲染比赛记录

### scene-gameover（结算）
- 领奖台（1/2/3名 + 第4名）
- 统计卡片：收益、余额、用时、赛道、最高分
- 段位赛：额外显示段位进度

### scene-ranked（段位赛）
- 段位图标 + 名称 + 星级进度
- 费用/奖励信息
- 对手预览
- 段位规则面板

### 弹窗
- `#modal`：单轮结算弹窗（胜/负）
- `#confetti`：胜利彩纸容器

### 装备规则
- **每个槽位只能装备1件**
- 装备/卸下在装备弹窗(比赛外)或内联装备面板操作
- `getCosmeticBonuses()` 计算所有已装备物品的总加成
- 返回 `{ speed, stamina, luck, sprint, dodge, burst, focus, warmup, recovery, chaos, _activeEffects }`

### 属性作用
| 属性 | 作用 |
|------|------|
| speed | 基础速度乘数 |
| stamina | 降低领先者体力消耗 |
| luck | 扩大幸运范围 |
| recovery | 加快落后者恢复 |
| focus | 降低抖动(更稳定) |
| burst | 增强爆发速度 |
| chaos | 对手动量波动增大 |
| sprint | 最后冲刺加强 |
| dodge | 抵抗减速事件 |

- 彩虹赛道：7色区域，每段有不同 zone 类型

### 地形加成

- `getTerrainMatchRatio(item)` 计算匹配比例（精确匹配=1.0，亲和匹配=部分）

## 14. 比赛引擎（Race Tick）

### 初始化 (`beginRaceAnimation`)

1. 计算每只乌龟基础速度 = `turtle.speed / 100`
2. 应用玩家装备加成(速度/动量/等)
3. 应用 AI 装备加成(段位赛)
4. 生成障碍物(每圈若干)
5. 生成铁人三项能力值
6. 重建道具按钮(显示技能按钮)
7. 启动 `requestAnimationFrame(tick)`

### Tick 循环（核心物理模型）

每帧(~60fps)对每只乌龟执行：

```
1. 基础速度 = baseSpeed
2. 动量平滑 (0.3 ~ 1.8)
   - 随机抖动 ±0.15/帧
   - 领先者体力消耗 → 动量下降
   - 落后者追赶加速
   - 最后冲刺(>80%进度) → 动量提升
3. 区域速度修正(铁人三项)
   - 游泳区速度 × 0.5~0.95
   - 蛙跳区速度 × 0.7~1.2
4. 爆发/减速 Timer
   - burstTimer > 0 → 速度 ×1.3
   - slowTimer > 0 → 速度 ×0.4
5. 幸运加成(道具)
6. 有效速度 = base × momentum × zone × burst/slow × lucky
7. 障碍物碰撞检测
   - 闪避概率 = dodgeBonus
   - 命中 → 减速 + 动画
8. AI 套装技能自动释放
   - CD就绪 + 进度>10% + 1%概率/帧
9. 玩家技能 CD 倒计时 + 按钮更新
10. 进度更新: progress += effectiveSpeed × 0.05
11. 圈数检测(每100进度=1圈)
12. 完赛检测(progress >= totalLaps × 100)
```

### 随机事件
- 平均每 ~5秒触发一次
- 11种事件类型(加速/减速/闪电/香蕉等)
- 目标选择：随机/领先者
- 触发视觉效果 + 音效

### 套装拖尾粒子
比赛中有 4件套的乌龟每帧生成主题粒子：
- 冰霜：蓝色雪花
- 烈焰：红橙火焰上升
- 暗夜：紫色星辰闪烁
- 龙皇：金色龙鳞
- 彩虹：七彩变色

### 解说系统 (`showCommentary`)
- 基于比赛进展自动生成解说气泡
- 触发条件：开赛、领先、接近、冲刺、区域切换等
- 乌龟表情气泡 (`showBubble`)：开心/难过/嘲讽

## 19. UI 渲染流程

### 主渲染 `render()`
```
1. 更新顶栏数据(金币/胜场/圈数)
2. 如果没有赛道元素 → 渲染下注/道具按钮 → return
3. 选龟阶段 → 渲染4个选龟卡片
4. 选赛道阶段 → 渲染赛道选择卡片
5. 比赛中/结束 → 更新4车道:
   a. 计算乌龟位置 = progress % 100 (转 left%)
   b. 渲染乌龟 SVG + 光环类 + 动画类
   c. 显示名称/进度/体力条
   d. 套装拖尾粒子
6. 渲染赔率/连胜信息
7. 内联装备面板(选龟阶段)
8. 更新按钮状态
```

### 按钮渲染
- `renderBetBtns()`：下注金额按钮，段位赛时隐藏
- `renderItemBtns()`：道具按钮 + 套装技能按钮 + 装扮按钮
  - 快速路径：按钮已存在时只更新 disabled/used 状态
  - 完整重建：首次 or 技能按钮状态变化时

## 附录：重要设计决策

1. **单文件架构**：便于直接打开运行，无需构建工具
2. **CSS 变量主题**：统一配色，方便整体调色
3. **纯代码音效**：Web Audio API 合成，无外部依赖
4. **套装技能手动触发**：避免自动特效导致赛道变形(之前 transform 动画会影响布局)
5. **CSS 特效仅用 filter/opacity**：不影响 layout 的视觉效果
6. **快速路径渲染**：`renderItemBtns` 等函数检测已有 DOM 元素时只更新状态，避免每帧重建
7. **进度模型**：0~(totalLaps×100)，每100=1圈，% 100得圈内位置
8. **动量模型**：模拟真实赛跑的加速/减速/体力消耗，让比赛有趣而非线性
