# 合成大乌龟 — 游戏设计文档

## 概述

一款基于「合成大西瓜」玩法的休闲合成游戏，主题替换为乌龟。玩家从顶部释放不同种类的乌龟，相同种类碰撞后合成为更高级的乌龟，最终目标是合成出加拉帕戈斯象龟。游戏以单个 HTML 文件交付，支持微信内分享和打开即玩。

## 目标平台

- 微信内置浏览器（主要）
- 移动端浏览器（通用）
- 竖屏优先，触摸操作

## 技术方案

- **单文件 HTML**：所有代码、样式、资源内嵌在一个 HTML 文件中
- **Canvas 渲染**：使用 HTML5 Canvas 绘制游戏画面
- **自写轻量物理引擎**：处理重力、圆形碰撞检测、碰撞响应、堆叠
- **Web Audio API**：程序化生成所有音效，零外部音频文件
- **零外部依赖**：无需加载任何 CDN 资源，离线可玩

### 视觉风格

- **卡通可爱风**：圆润造型、大眼睛、鲜艳配色
- 每种乌龟用 Canvas 绘制（椭圆壳体 + 圆形头部 + 椭圆四肢 + 尾巴）
- 每种有独特主色调，一眼可辨
- 壳体内部颜色比外部略深，增加层次感
- 最终级别（加拉帕戈斯象龟）带皇冠和金色光环

### 掉落规则

- 随机生成乌龟供玩家投放，具体范围由难度决定
- 高级乌龟（超出掉落范围的）只能通过合成获得
- 玩家可以看到当前乌龟和下一个乌龟的预览
- 在掉落范围内，各等级乌龟出现概率均匀随机

## 难度系统

游戏开始前可选择难度，影响随机掉落的乌龟等级范围：

| 难度 | 掉落范围 | 说明 |
|------|----------|------|
| 简单 | Lv.1-7（草龟~苏卡达陆龟） | 高级龟直接出现，合成链短，轻松上分 |
| 普通 | Lv.1-6（草龟~海龟） | 中等挑战 |
| 困难 | Lv.1-5（草龟~鳄龟） | 只出中小龟，全靠合成达到高级，挑战最大 |

设计逻辑：掉落等级范围越大 = 越容易直接获得高级龟 = 越简单。困难模式下合成链最长，需要更多策略性摆放。

## 游戏界面布局

竖屏移动端优先，从上到下依次为：

1. **顶栏**：当前分数（左）+ 游戏标题「🐢 合成大乌龟」（中）+ 最高分（右）
2. **难度选择器**：简单 / 普通 / 困难 三个按钮，游戏开始前可切换，游戏中锁定
3. **预览区**：显示「下一个」将出现的乌龟缩略图
4. **投放区**：当前待投放乌龟 + 向下虚线引导线，左右滑动控制水平位置
5. **游戏区**：深色渐变背景（#1a1a2e → #162447），乌龟在此堆叠和合成
6. **Game Over 线**：游戏区顶部附近的虚线，乌龟停留在此线以上超过 2 秒则游戏结束（短暂越线不触发，避免瞬间判定造成挫败感）
7. **底栏图鉴**：展示所有 9 种乌龟的缩略图，已合成过的点亮，未合成的灰色

## 交互设计

### 核心操作
- **左右滑动 / 拖动**：移动当前乌龟的水平位置
- **点击 / 松手**：释放乌龟，开始自由落体
- 释放后有短暂冷却时间（~0.5秒），防止连续快速投放

### 触摸适配
- 触摸事件（touchstart/touchmove/touchend）优先
- 同时支持鼠标事件作为桌面端回退
- 防止页面滚动干扰游戏操作（preventDefault）

## 物理引擎

自写轻量 2D 物理引擎，精确还原合成大西瓜的物理手感：

### 模拟循环
- 固定时间步长（fixedDeltaTime = 1/60秒），与渲染帧率解耦
- 每帧可能执行多次物理步进（accumulator 模式），确保不同设备上物理表现一致
- 每个物理步进内执行：施加重力 → 更新速度/位置 → 碰撞检测 → 碰撞求解 → 位置修正

### 重力系统
- 恒定重力加速度 `g ≈ 980 px/s²`（模拟真实重力感，可微调）
- 速度积分：`vy += g * dt`
- 位置积分：`y += vy * dt`

### 碰撞检测
- 所有乌龟视为圆形（碰撞半径见等级表）
- **圆-圆碰撞**：两圆心距离 < 两半径之和时判定碰撞
- **圆-边界碰撞**：游戏区左右墙壁和底部地面
- **宽相位优化**：当乌龟数量较多时，使用空间网格（spatial grid）加速碰撞对查找，避免 O(n²) 全量检测

### 碰撞求解（核心，还原合成大西瓜手感）

#### 位置修正（Positional Correction）
- 碰撞后首先分离重叠：计算穿透深度 `overlap = (r1 + r2) - distance`
- 按质量反比分配位移（质量与半径平方成正比）：大龟少动，小龟多动
- 加入少量分离偏移（bias ≈ 0.2~0.5），防止乌龟持续重叠抖动

#### 速度响应（Impulse Resolution）
- 沿碰撞法线（两圆心连线方向）计算冲量
- 恢复系数（restitution）≈ 0.3，碰撞后保留少量弹跳，不会像弹力球乱弹
- 冲量公式：`j = -(1 + e) * v_rel·n / (1/m1 + 1/m2)`
- 碰撞后沿法线分配速度变化：`v1 += (j / m1) * n`，`v2 -= (j / m2) * n`

#### 摩擦力
- 切线方向（垂直于碰撞法线）施加摩擦冲量
- 摩擦系数 ≈ 0.4，确保乌龟堆叠时不会无限滑动
- 使用库仑摩擦模型：切线冲量 ≤ μ * 法线冲量

#### 迭代求解
- 每个物理步进内，碰撞求解迭代 3~5 次（类似 Sequential Impulse Solver）
- 多次迭代确保堆叠的乌龟群稳定，不会穿透或抖动

### 合成逻辑
- 同级合成：两只相同等级乌龟碰撞时，标记待合成
- 合成在碰撞求解之后统一处理（避免求解过程中修改对象列表）
- 消除两者，在碰撞中点生成下一级乌龟
- 新龟继承两者的平均速度 * 衰减因子（~0.5），带有轻微向上的弹起效果
- 合成后新龟立即参与下一帧碰撞检测，可触发连锁合成

### 堆叠稳定性
- 速度阻尼：每帧对速度施加微小阻尼（damping ≈ 0.99），模拟空气阻力，帮助堆叠收敛稳定
- 速度低于阈值（< 0.5 px/s）且持续 0.5 秒的乌龟进入休眠状态
- 休眠乌龟跳过物理计算，节省性能
- 任何碰撞事件唤醒相关休眠乌龟及其接触链上的邻居

### 边界处理
- 左右墙壁：碰撞后水平速度反转 * restitution，同时位置修正到墙内
- 底部地面：碰撞后垂直速度反转 * restitution，同时位置修正到地面上
- 底部额外摩擦：地面摩擦系数更高（≈ 0.6），帮助乌龟快速稳定

## 特效系统

### 合成粒子特效
- 两龟合成时，在合成位置爆发 8-12 个圆形粒子
- 粒子颜色取新龟的主色调
- 粒子向外随机扩散，同时渐隐消失（~0.5秒）

### 分数弹出
- 合成位置弹出 "+XX" 分数文字
- 文字向上漂浮 + 渐隐（~0.8秒）

### 连锁合成
- 如果合成后新龟又触发下一次合成
- 粒子数量翻倍、分数文字加大
- 给玩家强烈的成就感和视觉反馈

### 最终合成庆祝
- 合成出加拉帕戈斯象龟（Lv.9）时
- 全屏金色粒子庆祝特效（~2秒）

### Game Over
- 屏幕轻微变暗（半透明黑色遮罩）
- 弹出结算面板：最终分数、最高分、重玩按钮、分享按钮

### 掉落引导线
- 当前乌龟正下方显示半透明虚线
- 帮助玩家精确瞄准投放位置

## 音效系统

使用 Web Audio API 的 OscillatorNode 程序化生成，零外部文件：

| 事件 | 音效描述 | 时长 |
|------|----------|------|
| 掉落 | 短促下降音，清脆的 "咚" | ~0.1秒 |
| 合成 | 上升音调 + 轻微回响，等级越高音调越高 | ~0.3秒 |
| 连锁合成 | 叠加和弦效果，越连锁越悦耳 | ~0.3秒 |
| 最终合成 | 胜利号角音效 | ~1秒 |
| Game Over | 低沉下降音 | ~0.5秒 |

- 无背景音乐，保持轻量，不打扰微信聊天环境

## 震动反馈

利用 Vibration API 增强触觉反馈：

- 普通合成：短震动 `navigator.vibrate(50)`
- 连锁合成：震动加长 `navigator.vibrate(100)`
- Game Over：长震动 `navigator.vibrate(200)`
- 不支持震动的设备自动跳过，无需特殊处理

## 计分系统

- 每次合成获得对应等级的分数（见等级表）
- 连锁合成分数叠加
- 最高分持久化存储在 localStorage
- Game Over 时显示本局分数和历史最高分

## 分享功能

- Game Over 结算面板提供「分享」按钮
- 点击分享按钮后，将分数文案复制到剪贴板（使用 Clipboard API），提示用户粘贴分享
- 文案示例：「🐢 我在【合成大乌龟】中获得了 XXX 分！最高合成到了 XX！来挑战我吧 →」

## 数据持久化

使用 localStorage 存储：

- `turtle_merge_high_score`：历史最高分
- `turtle_merge_difficulty`：上次选择的难度

## 性能目标

- 目标帧率：60fps
- 同屏最大乌龟数：~40个
- 文件总大小：< 80KB（单个 HTML 文件）
- 首次加载时间：< 1秒（微信内置浏览器）

## 兼容性

- iOS Safari 12+
- Android Chrome 70+
- 微信内置浏览器（X5/XWEB 内核）
- 需处理：安全区域适配、触摸事件兼容、Web Audio 自动播放限制（需用户首次交互后激活 AudioContext）
